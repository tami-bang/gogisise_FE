# Core Rules: Flow, Visibility & Learning

## 1. Drive the Code, but Report the Route (In Korean)
- You can write or update code freely to maintain the flow, but you MUST report exactly **which file** you modified and **what** changed.
- **CRITICAL: All explanations, responses, and reports must be written in Korean.**

## 2. Use Technical Terms with Real-World Analogies (Educational)
- I want to learn while vibe coding. Feel free to use standard technical jargon (e.g., State, Async, API, Component).
- **REQUIRED:** Whenever you use a technical term, you MUST provide a simple, real-world analogy or concrete example so a beginner can grasp the concept instantly.

## 3. Leave Visual Breadcrumbs (Comments in Korean)
- Always add short, simple comments right next to the new or changed code so I can visually spot your work in the file later.
- **Write these code comments in Korean.**

## 4. Checkpoint Summary
- After every task, give me a 1~2 sentence summary in Korean of what works now, and ask how I want to proceed next.
