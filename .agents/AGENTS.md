# Git Commit Rules

When you (the AI) make commits to this repository, you MUST follow these commit message rules:

## Types & Emojis
- 🎉 init: 프로젝트 초기 구조 생성
- ✨ feat: 새로운 기능 추가 (예: 시세 조회 화면 구현, 로그인 API 연동 등)
- 🐛 fix: 버그 수정 (예: 가격 계산 오류 수정, 모바일 레이아웃 깨짐 수정 등)
- ♻️ refactor: 코드 리팩토링 (예: 공통 컴포넌트 분리, API 호출 구조 개선 등)
- ⚡ perf: 성능 개선 (예: 이미지 최적화 등)
- 📝 docs: 문서 수정 (예: 기능 명세서 업데이트 등)
- 🔧 chore: 설정 변경, 의존성 추가 (예: Flutter 의존성 추가, Vite 설정 변경 등)
- 🚀 deploy: 배포 관련 (예: Vercel 배포 등)
- 🔥 remove: 사용하지 않는 파일/컴포넌트 삭제

## Principles
1. **한 커밋 = 한 작업**: 여러 작업을 하나의 커밋으로 묶지 않습니다.
2. **제목은 50자 내외로 간결하게**: 커밋 제목은 짧고 명확하게 작성합니다.
3. **현재형으로 작성**: 제목의 끝은 '추가', '수정', '분리', '구현' 등의 현재형 명사로 끝냅니다.
4. **커밋만 봐도 무엇을 했는지 알 수 있게 작성**: 직관적인 제목을 사용합니다.

## Body Format
상세 내용은 다음과 같이 하이픈(`-`)을 사용하여 무엇을 어떻게 고쳤는지 설명합니다:
```
- [수정/추가 사항 1]: [어떻게 고쳤는지에 대한 설명]
- [수정/추가 사항 2]: [어떻게 고쳤는지에 대한 설명]
```
