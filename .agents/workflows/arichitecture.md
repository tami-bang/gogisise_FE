## 1. 계층 분리 원칙 (Separation of Concerns)
에이전트는 `web/` 소스 코드를 생성하거나 수정할 때, 로직의 복잡도가 증가하더라도 다음 2가지 계층을 파일 및 함수 단위로 철저히 격리해야 한다.

### 1.1 도메인 및 데이터 계층 (Domain & Data Layer)
- **역할**: 순수 비즈니스 로직 연산, 데이터 포맷 가공, 저장소 제어 인터페이스.
- **구현 제약**:
  - 브라우저 `localStorage` 연동(setItem, getItem) 및 데이터 직렬화 함수는 UI 컴포넌트와 별개로 분리한다.
  - 향후 PostgreSQL DB 및 크롤러 API 연동 모듈이 추가될 때도 이 계층 하위에 격리하여 확장한다.
  - React 전역 상태(Context API)를 활용하여 앱 전체의 '단일 진실 공급원(SSOT)' 역할을 수행하게 한다.

### 1.2 프레젠테이션 계층 (Presentation Layer)
- **역할**: 사용자 화면 UI 렌더링 및 사용자 이벤트(터치/클릭) 트리거 수신.
- **구현 제약**:
  - JSX/HTML 내부에는 복잡한 비즈니스 연산 식이나 데이터 저장소 직접 접근 코드를 집어넣는 것을 전면 금지한다.
  - 오직 상위 도메인 계층에서 커스텀 훅이나 상태 Context를 통해 공급(상속)해 준 데이터와 핸들러 함수를 바인딩하여 렌더링하는 역할만 수행한다.

## 2. 확장 가이드라인
- 새로운 API나 외부 라이브러리(예: 카카오톡 SDK 등)가 도입되는 경우, UI 컴포넌트 내부에 직접 SDK 함수를 박지 말고 본 문서의 데이터 계층에 '서비스 모듈' 인터페이스 함수를 먼저 확보한 뒤 UI에서 이를 호출하게 한다.
