사용자가 채팅창에 `/structure`를 입력하면, 에이전트는 프로젝트 내부 파일들을 개별적으로 읽어서 분석하지 말고(토큰 낭비 방지), 반드시 아래의 2단계 워크플로우를 직접 실행하여 메모리를 갱신해야 한다.

## [실행 프로세스]

### 1단계: Tree Skill 실행 (디렉토리 구조 파악)
현재 OS 환경(macOS)에 맞는 터미널 명령어를 직접 실행하여 최상위 디렉토리 구조를 한눈에 파악한다.
- **실행할 명령어:** ```bash
  tree -L 4 -I "node_modules|.git" --dirsfirst
  ```

### 2단계: 프로젝트 디렉토리 구조 콘텐츠 생성
1단계에서 출력된 트리 결과를 바탕으로, 에이전트는 사용자가 직관적으로 이해할 수 있도록 **[출력 포맷]**을 엄격히 준수하여 최종 콘텐츠를 생성해야 한다.

---

## [출력 포맷]

에이전트는 사용자가 제시한 아래의 세 가지 요구사항을 반드시 포함하여 답변을 작성하고 `.agents/memory/structure.md`를 갱신한다.

### 1. 프로젝트 디렉토리 구조 (Project Structure)
본 문서의 목적이 현재 프로젝트의 최신 구조를 정의하고 동기화하는 작업임을 명시한다.

### 2. 전체 구조 트리 (주석포함)
1단계에서 획득한 트리 구조의 **모든 폴더와 개별 파일명 오른쪽에 공백(Tab 또는 Space)을 맞추어 정렬한 뒤, `#` 기호와 함께 핵심 역할/기능을 설명하는 한 줄 주석을 누락 없이 반드시 추가**한다.

* **정렬 및 주석 서식 예시 (반드시 이 형태를 따를 것):**
  ```text
  .
  ├── docs                                   # 프로젝트 관련 기획 및 디자인 문서(설계도) 보관
  │   ├── bible                              # 프로젝트 핵심 원칙 및 명세서(바이블)
  │   │   ├── FUNCTIONS.md                   # 기능 명세서
  │   │   └── PRD.md                         # 제품 요구사항 정의서
  │   └── design                             # 화면 설계 및 컴포넌트 디자인 시스템
  │       └── components                     # 재사용 가능한 UI 컴포넌트 명세
  │           ├── 1_button.md                # 버튼 컴포넌트 명세
  │           └── 2_card.md                  # 카드 컴포넌트 명세
  ├── .agents                                # 에이전트 시스템 전체 메인 가이드라인 (Entry Point)
  │   └── workflows                          # 명령어/태스크별 에이전트 작동 워크플로우
  │       └── structure.md                   # 구조 갱신 명령(/structure) 처리 워크플로우
  ├── src                                    # 실제 서비스가 동작하는 코드 메인 디렉토리 (공장 내부)
  │   ├── main                               # 제품을 생산하는 실제 애플리케이션 로직
  │   └── test                               # 제품에 불량이 없는지 검사하는 테스트 코드
  └── README.md                              # 인간 개발자를 위한 프로젝트 기본 설명서
  ```

### 3. 주요 폴더 및 파일 설명 (비유포함)
출력된 주요 디렉토리 영역(`docs`, `.agents`, `src` 등)의 유기적인 관계를 사용자가 쉽게 이해할 수 있도록 **'회사 조직', '기억 장치', 또는 '설계도/공장'과 같은 직관적인 비유(Metaphor)**를 들어 상세히 매핑하여 설명한다.

## [주의 사항]
- **누락 금지:** 하위 단에 있는 상세 설정이나 개별 파일(예: `1_button.md` 등)까지 단 하나도 빼놓지 말고 우측에 주석을 달아야 합니다.
- **토큰 절약:** 절대 소스 파일 내용을 하나하나 `cat`이나 `read`로 열어서 토큰을 낭비하지 마십시오. 오직 디렉토리 구조 정보와 기존 가이드라인의 맥락만을 결합하여 직관적인 콘텐츠를 생산하십시오.
