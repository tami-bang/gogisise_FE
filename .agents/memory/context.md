# 프로젝트 맥락 및 공통 로직 기억

## 💡 프로젝트 핵심 방향성
- **축산물(한우, 한돈) 시세 조회 서비스** (`gogisise_pjt`)
- **철저한 관심사 분리**: UI 컴포넌트는 데이터나 API 구조를 직접 하드코딩하지 않고, 무조건 `src/data/mockData.js`와 `src/api/priceApi.js`를 거쳐서 가져옵니다.
- **API 연동 대비**: 향후 백엔드 API가 완성되면 `priceApi.js` 내부만 수정하면 되도록 UI 계층과 API 계층을 느슨하게 연결해 둡니다.

## 🛠️ Utils 공통 로직 (UI 스타일 규칙)
각 화면(Pages)에서 텍스트 크기나 스타일을 다룰 때는 개별적으로 클래스를 지정하지 않고, 반드시 아래의 공통 유틸 함수를 사용합니다.

- **위치**: `src/ui/utils/textStyle.js`
- **핵심 유틸 함수**: `getTextClasses(size)`
- **기억할 점**: 온보딩 1단계 등에서 사용자가 선택한 텍스트 사이즈(State)에 따라 화면 전체의 글자 크기가 유기적으로 변경되어야 하므로, UI 컴포넌트 마크업 시 이 유틸 함수가 반환하는 클래스를 바인딩해야 합니다.
