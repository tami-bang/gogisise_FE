# 온보딩 Step 2 - 축종 선택 화면 명세

## 1. 화면 개요 및 구조 규칙
- **화면 목적**: 비즈니스 타겟에 맞춘 두 가지 메인 도매 축종 카테고리를 한 눈에 식별하고 오작동 없이 단일 선택하도록 가이드
- **비즈니스 가드레일**: 이 앱의 시세는 무조건 **'암컷' 데이터만 전제**하므로, 버튼 레이블 및 카드 명칭에 암컷 기준을 상시 고정 노출합니다.
- **시각적 경험**: 상단 Viewing Area 배경 레이어에 축산 마켓을 연상시키는 그래픽 소스를 10% 반투명(`opacity-10`) 처리하여 세련미를 확보합니다.

## 2. 컴포넌트 조립 구성

### 2.1 상단 상태 인디케이터 구역 - [Header 및 Body 상단 영역]
- **배경**: 반투명 마켓 일러스트 레이어 적용
- **인디케이터**: 메인 주황색 컬러의 두터운 둥근 바 컴포넌트 노출로 시선 집중 (`--color-primary` #f85029)
- **메인 레이블**: "조회하실 고기 종류를 선택해 주세요" (`text-title-xl` 24px, 700, `--text-strong` #1f1f1f)
- **서브 설명 레이블**: "자주 확인하시는 축종을 선택하면 맞춤 정보를 먼저 보여드립니다." (`text-body` 18px, 500, `--text-muted` #666666)

### 2.2 독립형 리스트 선택 카드 구역 - [Body 하단 영역]
- **컴포넌트**: 독립형 세로 단일 컬럼 리스트 형태의 `Category Card` 2개 수직 배치 (`space-16` 간격 적층, 가로 스크롤 및 복수 열 구조 절대 금지)
- **카테고리 카드 A (한우 암컷)**:
  - 스타일: 기본 카드 규칙 준수 (`radius-xl` 20px, 배경 `--color-surface` #ffffff, `shadow-soft`)
  - 내부 요소: 둥근 원형 컨테이너 내부 🐄 소 일러스트 아이콘 탑재 + 우측 메인 텍스트: "한우 암컷 시세보기" (`text-title` 22px, 700, `--text-strong` #1f1f1f) + 우측 끝 오른쪽 꺾쇠 화살표 컴포넌트 결합
- **카테고리 카드 B (한돈 암컷)**:
  - 스타일: 기본 카드 규칙 준수 (`radius-xl` 20px, 배경 `--color-surface` #ffffff, `shadow-soft`)
  - 내부 요소: 둥근 원형 컨테이너 내부 🐖 돼지 일러스트 아이콘 탑재 + 우측 메인 텍스트: "한돈 암컷 시세보기" (`text-title` 22px, 700, `--text-strong` #1f1f1f) + 우측 끝 오른쪽 꺾쇠 화살표 컴포넌트 결합
- **선택 활성화 상태 제어**:
  - 사장님이 카드를 터치하여 선택 완료 시, 테두리가 2px 두께의 메인 주황색(`--color-primary` #f85029)으로 즉각 전환되며, 카드 배경색은 연한 주황 틴트 톤(`#ffedea`)으로 반전되는 누름 활성화 피드백을 연동합니다.

### 2.3 최하단 선택 완료 처리 바 - [Footer 영역]
- **동작**: 화면 최하단 상시 고정 배치 (`fixed`, 높이 96px, z-index: 60)
- **최하단 완료 버튼**:
  - 미선택 비활성화 상태: 배경색 `--color-disabled` (#c7c7c7) 회색 처리, 터치 비활성(Disabled) 상태 가이드 유지
  - 선택 완료 활성화 상태: 메인 액션 주황색(`--color-primary` #f85029) 100% 활성화, 체크 아이콘과 함께 "선택 완료하기" 레이블로 시각 변환 (`text-body-lg` 20px, 600, 글자색 `--text-inverse` #ffffff)
