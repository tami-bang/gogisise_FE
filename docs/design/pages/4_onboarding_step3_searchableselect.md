# 온보딩 Step 3 - 관심 부위 설정 화면 명세 (Searchable Select)

## 1. 화면 개요 및 구조 규칙
- **화면 목적**: 크롤링을 통해 유입될 다량의 세부 암컷 부위 중 사장님이 매일 아침 보고 싶은 선호 부위들을 직접 검색 및 터치하여 설정하는 화면
- **컴포넌트 분기 핵심**: 데이터 후보군이 많으므로 단순 복잡한 라디오/체크박스 위젯을 나열하는 형태를 지양하고, **상단 상시 고정 검색바를 활용한 동적 필터링 컴포넌트 (Searchable Select)** 구조를 엄격히 적용합니다.

## 2. 컴포넌트 조립 구성

### 2.1 상단 상시 고정 검색 영역 - [Header 및 Body 최상단 영역]
- **동작**: 화면 상단에 영구 고정(Sticky/Fixed, z-index: 50)되며 아래 리스트 스크롤 동작에 구애받지 않는 지지대 영역 레이아웃
- **인디케이터 배지**: "3단계: 관심 부위 검색" (`text-label` 16px, 600, 배경 `#ffedea`, 글자 `--color-text-red` #d14734)
- **메인 안내 레이블**: "자주 보시는 부위를 검색해보세요" (`text-title-xl` 24px, 700, `--text-strong` #1f1f1f)
- **라이브 검색 인풋 필드 (Search Input)**:
  - 스타일: 높이 규격 56px 보장, `radius-lg` (16px), 테두리 두께 1px solid `--color-border` (#dddddd), Focus 시 `2px solid --color-primary` (#f85029) 확장 구조 준수
  - 내부 요소: 좌측 내부 🔍 돋보기 검색 아이콘 정렬 배치 + 플레이스홀더 상시 기준 문구 `"부위 이름 입력 (예: 등심, 삼겹살)"` 명시 (`text-body` 18px, `--text-light` #999999)

### 2.2 검색 결과 인터랙션 단일 열 리스트 구역 - [Body 영역]
- **동작**: 내부 수직 스크롤 활성화 (`overflow-y-auto`), 가로 스크롤 레이아웃 일절 배제
- **데이터 필터링**: 입력 조건에 매칭되는 부위 리스트 아이템들이 동적으로 필터링되어 수직 단일 열 카드 형태로 노출됩니다.
- **매핑용 암컷 샘플 데이터 리스트 구성**:
  - [한우 암컷 샘플]: 등심 (구이 및 스테이크용) | 안심 (가장 부드러운 부위) | 채끝 | 갈비 | 양지 | 사태 | 부채살
  - [한돈 암컷 샘플]: 삼겹살 (구이용 인기 대표 부위) | 목살 | 앞다리살 | 뒷다리살 | 항정살 | 갈비살 | 가브리살
- **카드 내부 선택 인터랙션**:
  - 각 부위는 대형 `Checkbox`가 결합된 `elevation-soft` 카드 형태로 매핑되어 고령 사장님의 터치 오작동을 원천 방지합니다.
  - 선택(체크) 완료 시, 우측 끝 원형 내부의 체크 아이콘이 활성화 상태로 색채 채움 변환을 일으키고, 카드 테두리는 메인 주황색(`--color-primary` #f85029)으로, 배경은 `#ffedea` 틴트로 색상 반전 피드백을 제공합니다.

### 2.3 최하단 완료 고정 영역 - [Footer 영역]
- **동작**: 백드롭 블러 또는 불투명 흰색 레이어가 적용된 상태로 화면 바닥에 상시 고정 (`fixed`, z-index: 60, 높이 96px)
- **완료 고정 액션 버튼**:
  - 스타일: 가로 너비 100%, 64px 높이 규격의 메인 주황 `Primary Button` 배치, 터치 누름 스케일 피드백 필수 탑재 (`active:scale-[0.98]`)
  - 레이블 명칭: "선택 완료하기" (`text-body-lg` 20px, 600, 글자색 `--text-inverse` #ffffff)
